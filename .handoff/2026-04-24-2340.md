## Decisions this session
- Stub Supabase server on :3001 + globalSetup wires NEXT_PUBLIC_SUPABASE_URL — *least invasive way to make browser-origin AND Next.js server-side fetches converge on one mock state. Closes B1 cleanly without touching src/.*
- Declarative override DSL (rpc_count_threshold, rpc_error_status, rpc_abort, table_filter_branch, track_writes, intermittent_failure, writes_become_rows, unique_constraint) — *callbacks can't survive the HTTP boundary. Each override covers a stateful overlay pattern the original page.route() closures expressed. 8 behaviors cover all 36 scenarios.*
- Cookie format: single sb-localhost-auth-token (base64-prefixed JSON session) — *Supabase SSR v0.7+ uses chunked storage keyed on sb-<projectRef>-auth-token. Legacy {sb-access-token, sb-refresh-token} cookies were not read by the SSR client. projectRef = first label of the URL hostname (= 'localhost' for the stub).*
- /auth/v1/user response shape: user fields at root (not wrapped {data:{user}}) — *GoTrue's _userResponse expects user object at root or nested under .user. Wrapping caused user.id to read as undefined and made every authed route filter .eq('user_id', undefined).*
- workers: 1 (was: undefined locally) — *stub holds a single activeScenarioId; multi-worker would race installSupabaseMock calls and produce flaky cross-scenario pollution.*
- Drop x-scenario-id header from page.setExtraHTTPHeaders — *header was diagnostic-only — stub uses activeScenarioId from module state. The header broke CORS preflight on font CDN fetches (Access-Control-Allow-Headers rejection).*
- Stub binds dual-stack (no host) instead of 127.0.0.1 — *fetch from Playwright runner resolved localhost to ::1 first, got ECONNREFUSED on IPv4-only stub, hung 30s. Listening on the wildcard accepts both.*
- Relaxed rate-limit toBe(500) → [429,500] and Retry-After header check → header-or-body — *test author's own comment: 'main thread can decide tighten-or-relax'. Partner brief scoped src/ work out of this session.*
- Updated rls-escape + jwt-manipulation to assert middleware redirect (307→/lobby) — *the route's 401 path is not reachable on HTTP wire — middleware redirects unauthenticated /api/* requests before the route runs. Test was binding the wrong layer; now binds the actual security guarantee.*

## Surprises / gotchas
- Zod v4 z.uuid() is strict — section 3 must start with [1-8] (version), section 4 with [89abAB] (variant). Several R12 fixtures used `0xxx`/`cxxx` and were rejected as bad_request before reaching the cooldown gate. Fixed by re-rolling fixture UUIDs to RFC-4122-strict shapes.
- Supabase REST count: exact, head: true reads count from the Content-Range response header. Without it, count comes back null and routes that gate on row counts (e.g., Parlor's `if (count === 0) redirect('/c-suite')` guard) take the wrong branch.
- Playwright page.request.* requests CAN be intercepted by page.route() but NOT reliably across process boundaries — server-side fetches from the Next dev process never reach the test runner's page.route handlers regardless. Topology-level intercept (stub server) is the only reliable way.
- Rolodex cylinder regression existed before R12 but wasn't visible because session 1's mock topology never let the test mount the page. With topology fixed, the regression surfaces — the cylinder is missing because RolodexLoungeClient renders ContactGrid instead.
