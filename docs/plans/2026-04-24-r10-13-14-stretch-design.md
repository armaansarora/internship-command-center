### 4.2 API contract

```
POST /api/offers/[id]/simulate
```

Authenticated (401 on miss) · owner-scoped offer lookup (404 on miss).

Request body:
```ts
{
  stance: {
    anchor: number,    // user's target base $
    flex: number,      // max concession $
    walkaway: number   // below this, walk
  },
  history: Array<{ role: "user" | "recruiter", text: string }>,
  userReply?: string   // omitted on round 0
}
```

Response:
```ts
{
  recruiterReply: string,
  scoring: null | {
    anchorScore: number,         // 0..5
    concessionScore: number,     // 0..5
    walkawayScore: number,       // 0..5
    critique: string             // one-line coaching, ≤240 chars
  },
  round: number,                 // server-computed, 0..4
  done: boolean                  // round >= 4
}
```

### 4.3 `simulator-turn.ts` (generateObject helper)

```ts
const SimTurnSchema = z.object({
  recruiterReply: z.string().min(10).max(1200),
  scoring: z.object({
    anchorScore: z.number().int().min(0).max(5),
    concessionScore: z.number().int().min(0).max(5),
    walkawayScore: z.number().int().min(0).max(5),
    critique: z.string().min(10).max(240),
  }).nullable(),
});

export async function simulateTurn(input: {
  userFirstName: string;
  offer: OfferRow;
  stance: { anchor: number; flex: number; walkaway: number };
  history: Array<{ role: "user" | "recruiter"; text: string }>;
  userReply: string | null;
}): Promise<SimulateTurnResult>
```

System prompt (inlined):

> You roleplay as a recruiter from `<offer.company_name>` negotiating the `<offer.role>` offer with `<userFirstName>`. You hold the CPO's calm, methodical voice (prep-room energy transferred into recruiter-chair energy) — direct, non-pushover, never mean. You know your offer and you push back reasonably when `<userFirstName>` counters.
>
> **Opening move (round 0):** open at a number that is **5–10% below** the actual offer base of $`<offer.base>` to create real anchor-holding pressure. Use round integers (e.g., if offer is $180,000, open $165,000–$170,000). State the number explicitly. Mention one soft-close lever (signing, equity refresh, start date).
>
> **Subsequent rounds:** respond to `<userFirstName>`'s counter. If they anchor strongly, concede modestly (5–10% of the gap). If they bend, don't reward it — hold or re-anchor. If they threaten to walk below your realistic floor, acknowledge and propose an alternate lever.
>
> **Private coaching rubric — do NOT reveal in your reply:**
> `<userFirstName>`'s targets are `anchor=$<stance.anchor>` / `flex=$<stance.flex>` / `walkaway=$<stance.walkaway>`.
>
> Score `<userFirstName>`'s **latest reply** (null scoring on round 0, where there is no user reply):
> - `anchorScore` 0–5: Did they re-assert a number near or above `stance.anchor`? (5 = strong re-anchor; 0 = capitulated silently)
> - `concessionScore` 0–5: If they conceded, was it ≤ `stance.flex`? (5 = no concession or within-flex; 0 = gave up >2× flex)
> - `walkawayScore` 0–5: If your last offer was below `stance.walkaway`, did they signal walk-away? (5 = well-timed; 0 = didn't signal despite below-walkaway)
> - `critique`: one-line coaching note, ≤240 chars.
>
> Rules:
> - No clichés ("I hope this email finds you well" is banned — recruiter speaks).
> - Short, direct replies. Recruiter ≤150 words per turn.
> - Never break the roleplay — no "As an AI" or tool references.
> - Integers in USD for any numbers you quote.

### 4.4 `NegotiationSimulator.tsx` (UI flow)

State:
```ts
type Phase = "stance" | "round" | "done";
const [phase, setPhase] = useState<Phase>("stance");
const [stance, setStance] = useState<Stance>({ anchor: 0, flex: 0, walkaway: 0 });
const [history, setHistory] = useState<Turn[]>([]);
const [turnScores, setTurnScores] = useState<Scoring[]>([]);
const [pendingReply, setPendingReply] = useState("");
const [loading, setLoading] = useState(false);
```

UI phases:

**`stance`** — three number inputs (anchor, flex, walkaway) + "Start simulation" button. Inputs default to `offer.base`, `Math.round(offer.base * 0.05)`, `Math.round(offer.base * 0.90)` as sane priors the user can override.

**`round`** — scroll log of `history`, each user turn followed by its `TurnScoreBadge`. Below the log: a textarea for the user's counter + "Send" button. Empty until the recruiter's opening lands. After round 4, transitions to `done`.

**`done`** — aggregate score row (mean of each axis), three `TurnScoreBadge`s showing turn-by-turn progression, "Start over" button that resets to `stance`.

Accessibility: `role="log"` on history, `aria-live="polite"` on new turns, labeled number inputs with `<label htmlFor>`.

### 5.2 API contract

```
POST /api/contacts/[id]/reference-request
```

Authenticated (401 on miss) · owner-scoped contact (404 on miss) · owner-scoped offer (404 on miss).

Request body:
```ts
{ offerId: string }  // uuid; required
```

Response:
```ts
{
  outreach: {
    id: string;
    subject: string;
    body: string;
    type: "reference_request";
    status: "pending_approval";
    ...  // full outreach_queue row
  }
}
```

Server-side flow:
1. Resolve contact by `id` under caller's `user_id`. 404 if absent.
2. Resolve offer by `offerId` under caller's `user_id`. 404 if absent.
3. Call `draftReferenceRequest({userFirstName, contact, offer})`.
4. Insert into `outreach_queue` with `type='reference_request'`, `status='pending_approval'`, `generated_by='cno'`, `metadata={offer_id, contact_id}`.
5. Return the inserted row.

### 5.3 `reference-request.ts` (generateObject helper)

```ts
const RefReqSchema = z.object({

export async function draftReferenceRequest(input: {
  userFirstName: string;
  contact: ContactForAgent;
  offer: OfferRow;
}): Promise<ReferenceRequestDraft>
```

System prompt (inlined, CNO Parlor-voice):

> You are the CNO of The Tower. You draft a reference-request email on behalf of `<userFirstName>` to `<contact.name>` about the offer from `<offer.company_name>`.
>
> **Rules:**
> - Warm, not formal. `<contact.name>` is a relationship, not a customer.
> - Specific. Name the company (`<offer.company_name>`) and role (`<offer.role>`). Reference any prior-interaction note from the contact record if present.
> - Clear ask: "Would you be open to serving as a reference — short phone call or written note?" (both options, user's pick).
> - End with: "No pressure if now isn't the right time" + offer to share more context on the role.
> - Max 180 words. Plain prose, no bullet lists.
> - **No clichés:** "I hope this email finds you well" is banned.
> - Sign off with `— <userFirstName>` — nothing more elaborate.

Prompt body: `CONTACT:\n<contactJson>\n\nOFFER:\n<offerJson>\n\nDraft the reference-request email.`
