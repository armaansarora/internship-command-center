# V2.0 Design: Internship Command Center

**Date:** 2026-03-09
**Status:** Approved
**Approach:** A — Turso + Auth.js + Vercel

## Problem

V1.0 is functional but bare bones. It works but doesn't feel like a real product. Key gaps:
- No notifications (actions silently succeed/fail)
- No page transitions (hard cuts between routes)
- No empty states, no loading polish
- Copy-paste workflow for emails (no Gmail integration)
- No calendar integration for interviews/follow-ups
- Local-only SQLite (can't access from phone or school)
- No networking/contact tracking
- Cover letters are one-shot (no versioning)

## Architecture

### Database: SQLite → Turso (Cloud SQLite)

```
Current:  better-sqlite3 → Drizzle → local SQLite file
V2.0:     @libsql/client → Drizzle → Turso cloud SQLite
```

Same Drizzle schema, same SQL dialect. Driver swap only. New tables added for:
- `accounts` — OAuth token storage for Auth.js
- `contacts` — Networking/referral tracking
- `cover_letter_versions` — Version history for generated letters
- `email_threads` — Synced Gmail threads linked to applications

Local dev uses Turso embedded replica (offline-capable, syncs on reconnect).

### Auth: Auth.js v5 + Google OAuth

Single OAuth flow provides:
- App authentication (whitelist: armaan.arora@nyu.edu, armaansarora20@gmail.com)
- Gmail API tokens (read + send)
- Google Calendar API tokens (read + create events)

JWT strategy with automatic token refresh. Scopes:
- `openid profile email`
- `https://www.googleapis.com/auth/gmail.readonly`
- `https://www.googleapis.com/auth/gmail.send`
- `https://www.googleapis.com/auth/calendar.events`
- `https://www.googleapis.com/auth/calendar.readonly`

### Deploy: Vercel

- Auto-deploy from GitHub on push
- Preview deploys per branch
- Environment variables: Turso, Google OAuth, Claude API, Tavily API
- Optional custom domain

## Features

### UI/UX Overhaul
- Framer Motion page transitions, list animations, stagger effects
- Toast notifications (sonner) for all mutations
- Command palette (cmdk — already installed, needs wiring)
- Loading skeletons on all data-fetching pages
- Empty states with CTAs for each page
- Inline table editing (status/tier directly in tracker)
- Mobile bottom tab bar, swipe actions on cards
- Micro-interactions: hover effects, press states, gradient tier badges

### Gmail Integration
- Gmail API messages.list with filters for application-related emails
- Match incoming emails to applications by company name/domain
- Unread application responses surfaced on dashboard
- Email thread view on application detail page
- Send follow-up emails directly from app (no copy-paste)
- Sent emails logged in activity feed

### Google Calendar Integration
- Auto-create calendar events for interviews
- Follow-up reminders as calendar events with notifications
- "Add to Calendar" button on follow-ups and interviews
- Upcoming events (next 7 days) shown on dashboard

### Smarter AI
- Interview prep: company overview, likely questions, talking points, recent news
- Cover letter version history: store every generation, compare side-by-side, mark favorites
- Company comparison: select 2-3 companies, table comparing culture/size/deals/fit
- Enhanced follow-ups: context-aware templates (thank-you vs. cold vs. referral nudge)

### Networking Layer
- New contacts table: name, company, email, phone, role, relationship type, warmth
- Contact cards on application detail pages
- "Who do I know at [Company]?" search
- Relationship warmth auto-decay (cold after 30 days no contact)
- Referral chain tracking (introduced_by foreign key)

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Turso over Supabase | Same SQL dialect as current SQLite. Driver swap, not schema rewrite. |
| Auth.js over custom OAuth | Battle-tested, handles token refresh, adapter ecosystem. |
| JWT over database sessions | Simpler for single-user. No session table needed. |
| Framer Motion over CSS-only | Orchestrated animations, exit animations, layout animations. |
| sonner for toasts | shadcn-compatible, accessible, minimal bundle. |
| Gmail API over IMAP | Google's recommended approach. OAuth tokens from Auth.js. |
| Contacts as separate table | Not every application has a contact. Many-to-many via company. |
| Vercel over Railway/Fly | Zero-config Next.js hosting. Preview deploys. Free tier sufficient. |
