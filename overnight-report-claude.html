06

Needs human review

17 real findings deliberately not auto-fixed — each touches an internal/external contract, DB concurrency, a DB-schema/RPC need, or a product decision where “smaller reversible change > big risky one.” This is the recommended next backlog.

COO briefing timezone

Renders interview times in server-UTC, not the user’s IANA tz (wrong-day for non-UTC users). Deferred: the function is a typed callback across many sites incl. the CEO dispatch-graph contract; wrong tz math is worse than consistent UTC.

Outreach duplicate-email on update failure

A failed status-update after a successful send re-sends next run. Deferred: email is an external contract; fix needs a Resend idempotencyKey.

getAgentMemories N+1 + lost-update race

Every chat retrieval fires N parallel access-count UPDATEs. Deferred: a race-free fix needs a new Postgres atomic-increment RPC (DB-schema change).

cio-reresearch per-row UPDATE loop

Batchable in principle. Deferred: relies on per-row error isolation + counter + notification — needs a partial-failure-handling decision.

Stripe webhook .single() on 0-row

A deleted profile (TOCTOU on checkout metadata) throws → 500 → multi-day Stripe retries. Deferred: the clean fix changes ack/retry/alerting semantics (a product/ops decision).

cfo-threshold cohort-maturity bias

Youngest cohort is structurally under-advanced → false “conversion fell” alerts. Deferred: needs an analytics redefinition or a status_history schema column.

owner-watchdog AI-cost 5000-row cap

Can under-count a runaway wave and stay “ok.” Deferred: needs server-side aggregation (RPC). Hard spend enforcement is separate & fail-closed.

draft-follow-ups timezone coverage

One daily 08:00-UTC run + a local [02:00,06:00) gate only sweeps ~UTC-6…-3. Deferred: widening needs more cron invocations or a window change (Hobby caps daily) — a product/deploy decision. In-code KNOWN-LIMITATION note added.

Dead code: calendar write helpers

createCalendarEvent/getUpcomingEvents have zero callers. Deferred: the write helper backs the OAuth write-scope request + a scope-minimization proof test — delete-and-tighten vs. wire-up is a product decision.

Dead code: getFloorIdentity accessor

Only its own test calls it; FloorShell reads the registry directly. Deferred: a “which pattern is canonical” decision.

sdk-poller archive self-wedge

A failed inbox→archive after enqueue wedges the file forever. Deferred: the fix touches the atomic-wx enqueue invariant.

match-delta non-atomic debounce

Two concurrent rescans can both run the (idempotent) rebuild. Deferred: LOW; the naive claim-first fix introduces a worse failure mode.

env() convention nits

A couple of optional-feature reads bypass EnvSchema. Deferred (low value): routing them through env() would turn a graceful-disable into a throw.

Bulk dead-code sweep (~3 dozen)

Many never-referenced exports. Deferred: a 30+-symbol removal is high-churn; best as a focused, individually-reviewed PR.

Cron time-budgets

A few batch crons sweep an unbounded user set with no maxDuration budget. Deferred: a scale-hardening task (budget + cursor) for deliberate design.

Gmail matcher substring breadth

Bidirectional includes could over-match. Deferred: pre-existing logic; wants its own test matrix before tightening.

negotiation-draft body lacks a Zod schema

Type-asserted (byte-bounded, token-DOS guarded). Deferred: a correct ParlorConveningResult schema is non-trivial; a wrong one rejects valid payloads.

Pre-existing (not a bug): the full npm test suite isn’t run in any CI workflow (only config+db subsets + non-blocking Playwright) — which is why the baseline tower-context failures were invisible. Recommended: a lightweight unit-tests.yml running tsc + test + lint per PR. Not added — CI workflow changes are owner-gated infra.
