04Reorganize — evaluated once, intentional no-op

The protocol calls for exactly one structural pass after fixes converge. The honest outcome here: don't move anything.

The repo already matches canonical Next.js 16 App-Router conventions — src/{app,components,lib,db,hooks,types,styles} with route groups (authenticated) / (marketing), an api/ tree, the standard root files (layout/page/error/loading/not-found/robots/sitemap/opengraph-image), proxy.ts (the Next 16 middleware rename), feature-colocated components, and colocated tests. 244 components, 931 lib modules, 104 route entries — all in their conventional homes, with an accurate living map in STRUCTURE.md. Moving 1,000+ files would be pure churn, and churn is a regression. 05Needs human review

Real findings deliberately left for a human — each touches an internal contract, an external service, or DB concurrency where an unattended fix would be riskier than the issue.

  • COO daily-briefing timezone (interviews shown in server-UTC, not the user's zone)
    Why deferred: getDailyBriefingData is a typed callback in the CEO dispatch-graph contract; threading a timezone is a wide change, and wrong day-boundary math shows the wrong interviews — worse than today's consistent UTC. Plan: optional timeZone? param, Intl-based local-midnight boundaries, wire the two direct callers first.
  • Outreach duplicate-email window on post-send status-update failure
    Why deferred: sending email is an external contract; the clean fix needs a Resend idempotency-key (verify SDK support) rather than a guess on the blast-brake state machine.
  • ArtLab sdk-poller archive self-wedge (rare)
    Why deferred: the fix touches the queue's atomic wx invariant; making enqueue idempotent globally could permit double-enqueue — worse than a rare wedge. Plan: a localized pre-enqueue existence check.
  • match-delta non-atomic debounce (duplicate idempotent rebuild)
    Why deferred: low severity (wasteful, not incorrect); the naive “claim-first” fix would skip retries for 5 min after a failed rebuild — a worse failure mode without racy compensation.
  • env() convention drift (a few process.env.* reads of declared vars)
    Why deferred (low value): routing optional-feature reads through env() couples them to full-schema validation — turning a graceful disabled-feature into a throw. Two undeclared flags were added to the schema as the safe subset.
  • Bulk dead-code sweep (~3 dozen unused exports)
    Why deferred: a 30+-symbol removal across many files is high-churn and risks removing something reached via a pattern static search misses (dynamic import, barrel re-export). The high-confidence ones were removed; the rest belong in a focused, individually-reviewed cleanup PR — not an unattended end-of-night sweep.
  • Cron time-budgets & a few low-severity tidy-ups
    A few batch crons sweep an unbounded user set with no maxDuration budget (scale-hardening, not a current outage); plus a Gmail substring-match tightening and a negotiation-draft Zod schema — both low-risk today and best authored deliberately.
  • CI gap (not a code bug)
    The full npm test suite isn't run by any workflow (only scoped subsets), which is why a failing test hid on main. Suggest a lightweight tsc + test + lint PR workflow — left to the owner since CI changes are infra-gated.
