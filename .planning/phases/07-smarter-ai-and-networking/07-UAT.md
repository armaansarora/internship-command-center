## Tests

### 1. Cold Start Smoke Test
expected: Kill any running dev server. Start the application fresh. Server boots without errors, homepage loads with live data.
result: pass

### 2. Contacts Page Navigation
expected: Click "Contacts" in the sidebar (or bottom tab bar on mobile). The /contacts page loads showing either a contacts table or an empty state if no contacts exist. Navigation item shows a Users icon.
result: pass

### 3. Add New Contact
expected: On /contacts page, click "Add Contact" button. A Sheet slide-out panel appears with fields (name, email, phone, company, relationship, notes, etc.). Fill in details and submit — toast confirmation appears and the new contact shows in the table.
result: pass

### 4. Warmth Badge on Contacts
expected: Contacts in the table show a color-coded warmth badge (Hot = green/emerald, Warm = amber, Cold = gray/zinc) based on when they were last contacted. Recently contacted contacts show "Hot", older ones show "Cold".
result: pass

### 5. Cover Letter Auto-Save to Version History
expected: Go to the Cover Letters page. Generate a cover letter for any application. After generation completes, switch to the "Version History" tab — the just-generated letter appears in the history, grouped under its company.
result: pass

### 6. Cover Letter Version Compare
expected: In Version History tab, expand a company group that has 2+ versions. Select two versions and click Compare. A side-by-side dialog opens showing both versions. You can also click "Set Active" on a version to mark it as the active one.
result: pass

### 7. Interview Prep on Detail Page
expected: Open an application detail page. An "Interview Prep" card appears (between Notes and Email History). Click "Generate" — structured sections appear: Company Overview, Likely Questions, Talking Points, Recent News. On revisit, the prep loads from DB without re-generating.
result: pass

### 8. Checkbox Selection & Company Comparison
expected: On the Applications tracker table, checkboxes appear in the first column. Select 2-3 applications by checking their boxes. A "Compare" button appears. Click it — a modal shows a structured comparison table (culture, size, deals, compensation, fit) generated by AI. A copy button lets you copy the data.
result: pass

### 9. Who Do I Know? Contact Cards
expected: On an application detail page, a "Who Do I Know?" section appears showing contacts associated with that company. Each contact card has a warmth-colored left border, warmth badge, and contact info (email/phone links). If no contacts exist at that company, the section is empty or hidden.
result: pass

### 10. Follow-Up Email Template Selector
expected: On an application detail page, open the Draft Email / Follow-Up section. A template type dropdown appears with 5 options (Follow-Up, Thank You, Cold Outreach, Referral Nudge, Post-Interview). If the application status is "interview", the template auto-selects "Thank You". Selecting a different template type changes the generated email tone.
result: pass
