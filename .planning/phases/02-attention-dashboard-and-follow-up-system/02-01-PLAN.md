# Phase 2 Plan 1: Attention Dashboard and Follow-Up System

<objective>
Build the attention dashboard as the landing page with ranked action items, status counters, and activity feed. Build the follow-up queue with auto-suggested timelines, snooze/dismiss/complete actions, and custom date setting. All Phase 2 requirements delivered in a single plan -- the original two-plan split was unnecessary because the dashboard and follow-up queue share the same data layer.
</objective>

## Context

Phase 1 delivered the application tracker with 71+ seeded applications, sortable/filterable table, detail views, and quick-add. The database schema already includes a `followUps` table. The landing page currently redirects to `/applications`. Phase 2 converts the landing page into an attention-first dashboard and adds a dedicated follow-up management queue.

## Tasks

### Task 1: Dashboard data layer (`src/lib/dashboard.ts`)

**Type:** auto

Build server-side query functions that compute dashboard data from the existing SQLite database:

- `getActionItems()`: Scans all applications and follow-ups to produce a ranked action item list. Priority order: (1) active interviews needing prep, (2) stale warm leads (in_progress with no update in 7+ days), (3) overdue follow-ups from the followUps table, (4) T1 applications with no response after 14+ days. Results sorted by priority, then by tier within each priority level.
- `getStatusCounts()`: Aggregates total applications and breakdown by status (applied, in_progress, interview, under_review, rejected, offer).
- `getRecentActivity()`: Returns last 10 updated applications ordered by `updatedAt` descending.

Typed interfaces: `ActionItem`, `StatusCounts`, `ActivityItem`.

### Task 2: Dashboard landing page and components

**Type:** auto

Convert the root page (`src/app/page.tsx`) from a redirect into the attention dashboard:

- **StatusCounters** (`src/components/dashboard/status-counters.tsx`): Grid of simple counter cards -- total, applied, in_progress, interview, under_review, offer, rejected. Color-coded per status. No charts or graphs.
- **ActionItems** (`src/components/dashboard/action-items.tsx`): Ranked list of up to 8 urgent items. Each shows company name, tier badge, reason text, and priority icon (Target for interview, Clock for stale lead, MessageSquare for overdue follow-up, AlertTriangle for T1 no-response). Each item links to the application detail page.
- **ActivityFeed** (`src/components/dashboard/activity-feed.tsx`): Compact list of recent status changes with relative timestamps (e.g., "3d ago") and status badges. Links to detail pages.
- **QuickAddForm** accessible from dashboard header (reuses Phase 1 component).
- Layout: status counters at top, 2-column grid below with action items (2/3 width) and activity feed (1/3 width).

### Task 3: Follow-up data layer (`src/lib/follow-ups.ts`)

**Type:** auto

Build query functions for the follow-up queue:

- `getPendingFollowUps()`: All follow-ups that are not completed and not dismissed, joined with application data (company, role, tier, status, contact info). Ordered by due date ascending.
- `getOverdueFollowUps()`: Subset of pending where `dueAt < now`.
- `getSuggestedFollowUps()`: Identifies active applications (not rejected, not offer) that have no pending follow-up, computes a suggested follow-up date using `suggestFollowUpDays(tier, status)`, and returns only those where the suggested date is in the past or within 3 days. Sorted: overdue first, then by tier.

### Task 4: Follow-up timeline suggestions (`src/lib/tier-utils.ts`)

**Type:** auto

Extract `suggestFollowUpDays(tier, status)` as a pure function in `tier-utils.ts` to avoid client components importing Node.js-dependent modules:

- Interview status: 1 day (post-interview thank-you)
- In-progress status: 5 days (warm lead check-in)
- T1 tier: 7 days
- T2 tier: 10 days
- Default (T3/T4): 14 days

### Task 5: Follow-up Server Actions (`src/lib/follow-up-actions.ts`)

**Type:** auto

Implement five Server Actions with Zod validation and `revalidatePath` for `/` and `/follow-ups`:

- `createFollowUp(formData)`: Insert new follow-up with applicationId, dueAt, optional note.
- `dismissFollowUp(formData)`: Set `dismissed = true` on a follow-up.
- `completeFollowUp(formData)`: Set `completedAt = new Date()` on a follow-up.
- `snoozeFollowUp(formData)`: Push `dueAt` forward by N days (1-30 range).
- `updateFollowUpDate(formData)`: Set a custom `dueAt` date on a follow-up.

### Task 6: Follow-up queue page and components

**Type:** auto

Build the `/follow-ups` page and its components:

- **Follow-ups page** (`src/app/follow-ups/page.tsx`): Shows overdue section (red header) and upcoming section, plus suggested follow-ups at the bottom. Empty state when no follow-ups exist.
- **FollowUpCard** (`src/components/follow-ups/follow-up-card.tsx`): Card for each pending follow-up showing company, tier badge, status badge, role, optional note, contact info, due date with relative timing ("3d overdue" or "In 5d"), and action buttons: Done, +3d snooze, +7d snooze, Dismiss. Red border when overdue.
- **SuggestedFollowUps** (`src/components/follow-ups/suggested-follow-ups.tsx`): Auto-suggested follow-ups with accept (creates the follow-up at suggested date) and dismiss buttons. Shows up to 5 suggestions with dashed border styling.
- **CreateFollowUp** (`src/components/follow-ups/create-follow-up.tsx`): Dialog-based form with date picker (defaulting to suggested date) and optional note field. Used on application detail pages.

### Task 7: Application detail page integration

**Type:** auto

Add a "Follow-Up" card to the application detail page (`src/app/applications/[id]/page.tsx`) containing the CreateFollowUp component, allowing users to schedule follow-ups directly from any application.

## Verification

- Dashboard is the landing page at `/` -- no redirect to `/applications`
- Action items display ranked by priority (interviews first, then stale leads, overdue follow-ups, T1 no-response)
- Status counters show correct totals matching the application data
- Activity feed shows last 10 updates with relative timestamps
- Quick-add button accessible from dashboard

- Suggested follow-ups appear for active applications without existing follow-ups
