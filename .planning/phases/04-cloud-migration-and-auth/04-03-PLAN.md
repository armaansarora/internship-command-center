   - Configure Google provider with ALL required scopes:
     ```
     scope: "openid email profile https://www.googleapis.com/auth/gmail.readonly https://www.googleapis.com/auth/gmail.send https://www.googleapis.com/auth/calendar.events https://www.googleapis.com/auth/calendar.readonly"
     ```
   - CRITICAL: Set `prompt: "consent"` and `access_type: "offline"` in authorization params — without these, Google will NOT return a refresh_token
   - Set `response_type: "code"`
   - Add `signIn` callback: check `profile.email` against `ALLOWED_EMAILS` env var (split by comma, trim whitespace). Return `false` if email not in whitelist.

If expired, refresh using Google's token endpoint (`https://oauth2.googleapis.com/token`) with `grant_type: "refresh_token"`. On refresh failure, set `error: "RefreshTokenError"`.
   - Add `session` callback: pass `token.access_token` as `session.accessToken` and `token.error` as `session.error`
   - Add TypeScript module declarations for `next-auth` (Session with accessToken, error) and `next-auth/jwt` (JWT with access_token, expires_at, refresh_token, error)
   - Configure `pages: { signIn: '/sign-in' }` to use custom sign-in page
   - Export `{ handlers, signIn, signOut, auth }`
   - Use the EXACT auth.ts code from the research doc Pattern 5, adding `pages` config

   IMPORTANT: If `proxy.ts` with the matcher pattern doesn't work in Next.js 16 (runtime error or routes not protected), fall back to per-page `auth()` checks. But try proxy.ts first — it's the intended pattern.
