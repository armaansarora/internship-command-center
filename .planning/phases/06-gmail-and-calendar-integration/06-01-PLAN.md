Create `src/lib/google.ts` with a `getGoogleClient()` async factory function:
- Import `auth` from `@/auth` and `google` from `googleapis`
- Call `await auth()` to get session
- If no `session?.accessToken`, throw "Not authenticated"
- If `session.error === "RefreshTokenError"`, throw "Token refresh failed"
- Create `new google.auth.OAuth2()`, call `.setCredentials({ access_token: session.accessToken })`
- Return `{ gmail: google.gmail({ version: "v1", auth: oauth2Client }), calendar: google.calendar({ version: "v3", auth: oauth2Client }) }`

Create `src/lib/gmail.ts` with these functions:

1. `parseEmailHeaders(message: gmail_v1.Schema$Message)` helper:
   - Extract From, To, Subject, Date, Message-ID, References headers from `message.payload?.headers`
   - Return `{ id, threadId, from, to, subject, date, snippet, isUnread }` where isUnread = `message.labelIds?.includes('UNREAD')`

2. `extractCompanyDomain(companyName: string): string` helper:
   - Heuristic: lowercase, remove spaces/special chars, append ".com"
   - Handle common patterns: "Goldman Sachs" -> "goldmansachs.com", "JP Morgan" -> "jpmorgan.com"
   - Return domain string

3. `searchCompanyEmails(gmail, companyName: string, maxResults = 10)`:
   - Build Gmail query: `from:@${domain} OR subject:${companyName}`
   - Call `gmail.users.messages.list({ userId: "me", q: query, maxResults })`
   - For each message ID returned, call `gmail.users.messages.get({ userId: "me", id, format: "metadata", metadataHeaders: ["From", "To", "Subject", "Date", "Message-ID", "References"] })`
   - Parse each with `parseEmailHeaders()` and return array

4. `getUnreadApplicationEmails(companyNames: string[], maxPerCompany = 3)`:
   - Call `getGoogleClient()` to get gmail instance
   - For each company, build query: `(from:@${domain} OR subject:${companyName}) is:unread newer_than:30d`
   - Use `gmail.users.messages.list()` then `gmail.users.messages.get()` with format "metadata"
   - Return array of `{ company: string, emails: ParsedEmail[] }` grouped by company
   - Wrap ALL Gmail API calls in try/catch — return empty array on error (graceful degradation)
   - Limit to max 5 companies searched to avoid API call explosion

Export types: `ParsedEmail` (the parsed header result type), `CompanyEmails` (company + emails grouping).

IMPORTANT: All functions must handle errors gracefully — catch and return empty results, never throw to caller. The dashboard must work even if Gmail is unreachable.
