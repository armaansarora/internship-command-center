Extend `src/lib/gmail.ts` with:

1. `getEmailBody(message: gmail_v1.Schema$Message): string` helper:
   - Traverse `message.payload.parts` recursively for multipart messages
   - Find part with mimeType "text/plain" (prefer) or "text/html" as fallback
   - Decode `part.body.data` from base64url: `Buffer.from(data, 'base64url').toString('utf-8')`
   - If no parts (simple message), use `message.payload.body.data`
   - Return decoded text content (or empty string if nothing found)

2. `getFullEmailThread(companyName: string, maxResults = 20)`:
   - Call `getGoogleClient()` to get gmail
   - Build query: `from:@${domain} OR to:@${domain} OR subject:${companyName}`
   - Call `gmail.users.messages.list()` then for each message `gmail.users.messages.get()` with format "full" (need body)
   - Parse each with `parseEmailHeaders()` and also extract body via `getEmailBody()`
   - Return array of `FullEmail` type (ParsedEmail + `body: string`)
   - Sort by date ascending (oldest first, like a conversation)
   - Wrap in try/catch, return empty array on failure

3. `sendEmail(to: string, subject: string, body: string, threadId?: string, inReplyTo?: string, references?: string)`:
   - Call `getGoogleClient()` to get gmail
   - Construct RFC 2822 message with headers: To, Subject, Content-Type (text/plain charset=utf-8), and optionally In-Reply-To and References for threading
   - Base64url encode: `Buffer.from(message).toString('base64').replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '')`
   - Call `gmail.users.messages.send({ userId: "me", requestBody: { raw: encoded, threadId } })`
   - Return `{ success: true, messageId: res.data.id }` on success, `{ success: false, error: string }` on failure

Export new type: `FullEmail` (extends ParsedEmail with body field).
