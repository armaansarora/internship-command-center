    1. Create `src/components/shared/swipeable-card.tsx`:
       - 'use client' directive
       - Follow the pattern from 05-RESEARCH.md Pattern 4 (Swipeable Card) with enhancements:
       - Import `motion, useMotionValue, useTransform` from `motion/react`
       - Props: `children`, `onSwipeLeft?: () => void`, `onSwipeRight?: () => void`, `leftLabel?: string` (default "Dismiss"), `rightLabel?: string` (default "Complete")
       - Use `drag="x"` (NOT `drag={true}`) to lock to horizontal axis and prevent scroll interference
       - Add `dragDirectionLock` prop to motion.div — this locks gesture to one axis once direction is determined, preventing scroll conflicts
       - `dragConstraints={{ left: -120, right: 120 }}`, `dragElastic={0.1}`
       - `style={{ x, opacity }}` where x is useMotionValue(0) and opacity transforms [-100,0,100] to [0.5,1,0.5]
       - `onDragEnd`: if offset.x < -80 call onSwipeLeft, if offset.x > 80 call onSwipeRight
       - Add visual indicators behind the card showing action labels:
         - Position relative wrapper
         - Behind the card (absolute positioned), show `leftLabel` on the right side (red-ish) and `rightLabel` on the left side (green-ish)
         - These labels become visible as the card is dragged
       - After callback fires, animate the card fully off-screen: use `animate` controls to slide card to x: -300 or x: 300 before calling the callback
       - Snap back if threshold not reached (dragConstraints handles this via elastic)

    2. Modify `src/components/follow-ups/follow-up-card.tsx`:

       - In the FollowUpCard component, detect mobile:
         ```typescript

         ```
       - Conditionally wrap the card content:
         - On mobile: wrap the existing card div in `<SwipeableCard onSwipeLeft={() => handleAction(dismissFollowUp)} onSwipeRight={() => handleAction(completeFollowUp)} leftLabel="Dismiss" rightLabel="Done">`
         - On desktop: render the card as-is (no swipe wrapper)
       - The card already has the button bar at the bottom — this remains visible on both mobile and desktop. Swipe is an ADDITIONAL interaction method, not a replacement.
       - Use a simple conditional: `const Wrapper = isMobile ? SwipeableCard : React.Fragment` pattern, or a straightforward if/else in the JSX.
