  <action>
    Add `import { toast } from 'sonner'` to each client component that calls a server action. Wrap each server action call with toast feedback based on the `{ success, error }` return pattern. Specifically:

    1. **quick-add-form.tsx** — After `createApplication(formData)` returns:
       - On error: `toast.error('Failed to add application', { description: result.error })`
       - On success: `toast.success('Application added')`

    2. **follow-up-card.tsx** — The `handleAction` function calls completeFollowUp/dismissFollowUp, and `handleSnooze` calls snoozeFollowUp:
       - After `completeFollowUp`: `toast.success('Follow-up completed')`
       - After `dismissFollowUp`: `toast.success('Follow-up dismissed')`
       - After `snoozeFollowUp`: `toast.success('Snoozed ${days} days')`
       - If any returns error: `toast.error('Action failed', { description: result.error })`
       - Note: These server actions currently don't capture the return value. Update `handleAction` and `handleSnooze` to capture `const result = await action(fd)` and check `result.error`.

    3. **create-follow-up.tsx** — After `createFollowUp(formData)` returns:
       - On error: `toast.error('Failed to create follow-up', { description: result.error })`
       - On success: `toast.success('Follow-up scheduled')`

    4. **cover-letter-generator.tsx** — After `generateCoverLetterAction(appId)` returns:
       - This action returns `GenerationState` not `{ success, error }`. Check if `result.error` exists:
       - On error: `toast.error('Generation failed', { description: result.error })`
       - On success (result.coverLetter exists): `toast.success('Cover letter generated')`

    Also check the application detail page (`src/app/applications/[id]/page.tsx`) for any inline server action calls (updateApplicationStatus, updateApplicationNotes) and add toast there too if they exist in client sub-components.

    Do NOT use `toast.promise()` — keep it simple with explicit success/error calls after the action resolves. This matches the existing sync-then-refresh pattern.

    Use explicit `id` parameter on rapid-fire toasts (e.g., status changes) to prevent toast flooding: `toast.success('Status updated', { id: 'status-update' })`.
