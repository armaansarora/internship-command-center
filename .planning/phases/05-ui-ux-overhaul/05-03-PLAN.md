  <action>
    Modify `src/components/applications/columns.tsx` to make the status and tier columns inline-editable using Select dropdowns.

    1. First check if `@/components/ui/select.tsx` exists (it should — shadcn select was installed). If not, run `npx shadcn@latest add select --yes`.

    2. Add imports at the top of columns.tsx:
       ```typescript
       import { Select, SelectContent, SelectItem, SelectTrigger } from '@/components/ui/select';
       import { updateApplicationStatus } from '@/lib/actions';
       import { toast } from 'sonner';
       import { STATUS_LABELS } from '@/types';
       import type { Status, Tier } from '@/types';
       ```

    3. Replace the **status column** cell renderer:
       - Instead of `<StatusBadge status={...} />`, render a Select dropdown:
       ```tsx
       cell: ({ row }) => (
         <Select
           value={row.original.status}
           onValueChange={async (value) => {
             const formData = new FormData();
             formData.set('id', String(row.original.id));
             formData.set('status', value);
             const result = await updateApplicationStatus(formData);
             if (result.error) {
               toast.error('Failed to update status', { description: result.error });
             } else {
               toast.success(`Status: ${STATUS_LABELS[value as Status]}`, { id: 'status-update' });
             }
           }}
         >
           <SelectTrigger
             className="h-7 w-[130px] border-none bg-transparent hover:bg-accent/50 focus:ring-0"
             onClick={(e) => e.stopPropagation()}
           >
             <StatusBadge status={row.original.status as Status} />
           </SelectTrigger>
           <SelectContent onClick={(e) => e.stopPropagation()}>
             {Object.entries(STATUS_LABELS).map(([value, label]) => (
               <SelectItem key={value} value={value}>{label}</SelectItem>
             ))}
           </SelectContent>
         </Select>
       )
       ```

    4. Replace the **tier column** cell renderer with the same pattern:
       - Create or import a server action for updating tier. Check if `updateApplicationStatus` can handle tier updates, or if a new `updateApplicationTier` action is needed.
       - If no tier update action exists, create one in `src/lib/actions.ts`:
         ```typescript
         export async function updateApplicationTier(formData: FormData) {
           const id = Number(formData.get('id'));
           const tier = formData.get('tier') as string;
           await db.update(applications).set({ tier }).where(eq(applications.id, id));
           revalidatePath('/applications');
           return { success: true };
         }
         ```
       - Tier Select with TierBadge as trigger display, T1/T2/T3/T4 as options
       - Use `toast.success('Tier: ${value}', { id: 'tier-update' })` with explicit toast id

    5. CRITICAL: Both SelectTrigger and SelectContent must have `onClick={(e) => e.stopPropagation()}` to prevent the table row's onClick handler from firing and navigating away. Also add `onPointerDown={(e) => e.stopPropagation()}` on the SelectTrigger for extra safety.

    6. The `useRouter` import and `router.refresh()` are NOT needed here because the server actions already call `revalidatePath`. However, the table data won't update visually until the page re-fetches. Since these are server components that pass data down, the revalidation will trigger a server re-render. If the UI doesn't update, add `router.refresh()` after the server action call. But try without first.

    Note: columns.tsx is already 'use client'. The Select onValueChange handler calls the server action directly — this works because server actions can be called from client components.
