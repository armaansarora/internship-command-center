<task type="auto">
  <name>Task 1: Create data migration script</name>
  <files>internship-command-center/scripts/migrate-to-turso.ts</files>
  <action>
    Create scripts/migrate-to-turso.ts that:

    1. Import createClient from @libsql/client and drizzle from drizzle-orm/libsql
    2. Import all schema tables from ../src/db/schema
    3. Create two DB connections:
       - localDb: drizzle(createClient({ url: 'file:./data/internship.db' }))
       - prodDb: drizzle(createClient({ url: process.env.TURSO_PROD_URL!, authToken: process.env.TURSO_PROD_TOKEN! }))
    4. Migrate tables in FK dependency order to avoid constraint violations:
       a. applications (no FK dependencies)
       b. companyResearch (no FK dependencies)
       c. followUps (references applications.id)
       d. contacts (self-referential FK -- insert those with introducedBy=null first, then update introducedBy for referral chains)
       e. coverLetters (references applications.id)
       f. interviewPrep (references applications.id)
    5. For each table:
       - Select all rows from localDb
       - If rows exist, insert into prodDb using db.insert(table).values(rows)
       - Log count: "Migrated N {tableName} rows"
       - Use batched inserts if > 100 rows (Turso has request size limits)
    6. Wrap in async main() with try/catch, log success/failure
    7. Add a "scripts" entry to package.json: "migrate:turso": "npx tsx scripts/migrate-to-turso.ts"

    Important: The script should be idempotent-safe. Add a --force flag or warn if production DB already has data (check applications count first). Default behavior: skip if data exists, proceed with --force.
  </action>

  </verify>
